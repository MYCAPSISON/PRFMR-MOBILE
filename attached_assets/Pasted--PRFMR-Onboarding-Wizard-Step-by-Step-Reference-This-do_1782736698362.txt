# PRFMR — Onboarding Wizard: Step-by-Step Reference

This document is the **exact** reference for replicating the PRFMR onboarding wizard.
Copy every heading, bullet, option label, and placeholder verbatim.
Every step number below is the **internal** `step` integer used in `onboarding.tsx`.

**No card. No progress bar (progress = 0%). No logo. No nav.**

Full-screen layout:
- Outer container: `bg-[#0a0a0a]`, `height: 100dvh`
- Image container: `width: min(100%, calc(100dvh × 863 / 1665))`, `aspect-ratio: 863 / 1665`
- Image: `BD246D36-3AE7-426E-88E2-322E8390F5AB_1777392079503.jpeg`, `object-fit: fill`, fills the entire container exactly — **no cropping, no letterbox**

**CTA — transparent hit-target overlay:**
The image has a button baked into the artwork. The invisible `<button>` is absolutely positioned on top of the artwork's button using image-relative percentages:
- `bottom: 1.8%` — `30px / 1665px`
- `height: 5.7%` — `95px / 1665px`
- `left: 2.3%`, `right: 2.3%` — `20px / 863px`
- Style: `border-0 outline-none bg-transparent cursor-pointer`
- `data-testid="button-splash-cta"`

## Step 23 — Projected Cut Chart (Skipped if nonFightPrepMode)

**Heading:** `Your projected cut.`
**Subheading:** `Based on the plan you just built.`

**Engine:** Calls `calculateWeightCutPlan()` again with same inputs as step 22. Pure SVG + CSS chart — no charting library.

### Chart construction

1. Build `trend` array: `[{ weight: current, label: "Today" }, ...up to 5 sampled interior weekly targets..., { weight: target, label: "Fight" }]`
2. Interior points: sampled evenly from `plan.weeklyTargets` (at most 5 points). Each gets a date label like `"14 Jul"`.
3. `toBarH(w) = round(((w − minW) / range) × 68 + 14)` — bar height as a 14–82% range
4. SVG `viewBox="0 0 100 100"`, x = `((i + 0.5) / trend.length) × 100`, y = `100 − toBarH(w)` (inverted)

### Rendering

**Background bars** (one per trend point):
- Last bar: `bg-primary/40`; others: `bg-primary/15`
- Framer Motion `initial={{ height: 0, opacity: 0 }}` → `animate={{ height: "{h}%", opacity: 1 }}`
- Bars are absolutely positioned bottom-aligned

**SVG polyline overlay** (on top of bars, `preserveAspectRatio="none"`):
- Stroke: `rgb(249,115,22)` (orange), `strokeWidth="2.5"`, `vectorEffect="non-scaling-stroke"`, `fill="none"`
- Each point is a `<circle r="3">` — last point: `fill="rgb(249,115,22)"`, others: `fill="#0a0a0a" stroke="rgb(249,115,22)"`

**Weight labels** (below each bar):
- Last: `text-primary font-bold text-[11px]`
- Others: `text-muted-foreground text-[11px]`

**Date labels** (below weight labels, `text-[9px] text-muted-foreground/60`)

**Stats row below chart (2 columns):**
- Left: `−{totalDrop.toFixed(1)} kg` projected loss, label `Projected loss`
- Right: `{weeklyRate.toFixed(2)} kg` per week, label `Per week (fat loss phase)`

### Live Readiness Score (updates instantly)

**Formula:**
```
sleepScore   = hours >= 8 ? 1 : hours >= 7 ? 0.8 : hours >= 6 ? 0.5 : 0.25
energyScore  = (demoEnergyLevel − 1) / 4      // maps 1–5 → 0–1
readiness    = round((sleepScore × 0.5 + energyScore × 0.5) × 100)
```

**Labels and colours:**

| Score | Label | Colour class |
|---|---|---|
| ≥ 80 | Excellent | `text-green-400` |
| ≥ 60 | Good | `text-primary` (orange) |
| ≥ 40 | Fair | `text-yellow-400` |
| < 40 | Low | `text-destructive` |

- Left: `Session Readiness` (xs muted) + label (`text-xl font-bold`)

### Tab: Search

**API:** `GET /api/foods/search?q={query}` — debounced 450 ms, fires when query ≥ 2 chars.

Search input with `Search` icon left, `placeholder="Search foods…"`, `data-testid="input-food-search"`

**States:**

- Loading: spinning `Loader2` + `Searching…`
- No results (after search): `No results for "{query}".`
- Results: list of food cards

**Result card:**
- Left: 40×40 product image (or `Search` icon placeholder if no image)
- Middle: food name (truncated) + `{brand} · {kcal} kcal · {protein}g P · {carbs}g C · {fat}g F`
- Right: toggle button — 28×28 circle, orange `Plus` or white `Check`

Added items tracked in `demoAddedFoods[]` as `"off-{index}"` strings. Nothing persisted to DB.

### Tab: Whole Food

**API:** `GET /api/ingredients` — lazy-loaded when this tab (or Barcode) first opens. Not re-fetched.

Search input with `Search` icon, `placeholder="Search whole foods…"`, `data-testid="input-whole-food-search"`

Loading spinner while ingredients list empty.

**Filter + sort:** client-side, `ing.name.toLowerCase().includes(query)`, sorted by `commonScore` descending, capped at 30 (empty query) or 50 (with query).

**Each ingredient row** (full-width button, `border rounded-lg px-3 py-2.5`):
- Name (`text-[13px] font-medium`)
- Macro line: `{kcal} kcal · {protein}g P · {carbs}g C · {fat}g F per 100g` (`text-[11px] muted`)
- Micro badges: top 3 non-zero micros from priority list below, shown in `text-[10px] text-emerald-500/80`

**Micro priority order (first 3 non-zero values shown):**
Omega-3 → B12 → Vit D → Vit C → Calcium → Iron → Selenium → Zinc → Magnesium → Potassium → Folate → B6 → Niacin → Vit A → B1 → B2

Added items go into `wholeAddedItems[]` — drives progress bar totals.
Added: `border-primary/50 bg-primary/10`, circle = `bg-primary` with white `Check`
`data-testid="button-whole-food-{index}"`

### Tab: Barcode

**API (after successful scan):** `GET /api/food/barcode/{barcode}`
**Also lazy-loads:** `GET /api/ingredients` (same as Whole Food tab)
**Scanner library:** `html5-qrcode` (dynamic import)

**Idle state (no scan in progress, no result):**
- Dashed circle placeholder with `Camera` icon (10×10, `border-primary/30`)
- Text: `Scan a barcode` / `Point your camera at any food packaging`
- Button: `Open Camera` (with `Camera` icon), `data-testid="button-start-scan"`

**Scanning state:**
- `<div id="reader-onboarding">` renders scanner viewport
- `Cancel` ghost button below, `data-testid="button-stop-scan"`
- Formats supported: EAN-13, EAN-8, UPC-A, UPC-E, CODE-128

**Loading state (after scan, looking up product):**
- `Loader2` + `Looking up product…`

**Result state (product found):**
- Product image (14×14) + name + brand
- Gram amount input with preset buttons: `50` `100` `150` `200`

## SportBadge Engine

### What it is

`<SportBadge mainSport="{level} {sportDisplay}">` renders a pill badge on the dashboard, profile, and in the onboarding badge preview. It uses **sport icon PNGs** (white icons, inverted with CSS filter) and detects the competition level from the string prefix.

### Badge construction

The `mainSport` string is built by `buildMainSport(level, sport)`:

```
buildMainSport("Pro", "Boxing")       → "Pro Boxer"
buildMainSport("Amateur", "Wrestling")→ "Amateur Wrestler"
buildMainSport("Semi-pro", "BJJ")     → "Semi-pro BJJ"
```

Where `sportDisplay(sport)` maps raw sport values to display names:

| Raw sport value | Display name |
|---|---|
| `Boxing` | Boxer |
| `MMA` | MMA |
| `Muay Thai` | Muay Thai |
| `Kickboxing` | Kickboxer |
| `BJJ` | BJJ |
| `Wrestling` | Wrestler |
| `Traditional martial arts` | Martial Artist |

### Level detection (from the string prefix)

```
starts with "Pro "      → "pro"    → amber/gold style
starts with "Amateur "  → "amateur"→ orange/primary style
anything else           → "custom" → slate/grey style
```

### Three visual styles

**Pro** (`level === "pro"`)
```
bg-gradient-to-r from-amber-500/25 to-yellow-400/15
border-amber-400/50
text-amber-300
```

**Amateur** (`level === "amateur"`)
```
bg-primary/15
border-primary/30
text-primary
```

**Custom** (any other prefix — e.g. "Semi-pro", "White collar")
```
bg-gradient-to-r from-slate-400/20 to-slate-300/10
border-slate-300/40
text-slate-200
```

### Sizes

| Prop | Padding | Font size |
|---|---|---|
| `md` (default) | `px-3.5 py-1.5` | `text-xs` |
| `sm` | `px-2.5 py-1` | `text-[11px]` |

All: `rounded-full inline-flex items-center gap-1.5 font-semibold border`

### Icon lookup

Icons are PNG files rendered as `<img h-5 w-5>` (md) or `<img h-4 w-4>` (sm).

**CSS filter applied to all icons:**
```css
filter: invert(1) brightness(2);
opacity: 0.9;
```
This makes them appear white regardless of the source PNG colour.

**Icon files** (all in `PRFMR Theme/sport-icons/`):

| Display name matched | Icon file |
|---|---|
| `Boxer` | `boxing.png` |
| `MMA` | `mma.png` |
| `Muay Thai` | `muay-thai.png` |
| `Kickboxer` | `kickboxing.png` |
| `BJJ` | `bjj.png` |
| `Wrestler` | `wrestling.png` |
| `Martial Artist` | `traditional.png` |

**Match algorithm:** case-insensitive partial match. e.g. `"pro boxer".includes("boxer")` → boxing icon.

### Where it appears

- Onboarding step 5: live badge preview as user types level
- Dashboard hero card
- Profile page
- `size="sm"` used in compact contexts (sidebar, lists)
